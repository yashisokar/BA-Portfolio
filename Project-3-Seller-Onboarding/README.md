# Project 3 — Seller Onboarding Portal Case Study (ShopEasy)

## Overview
End-to-end Business Analysis case study for a 
Seller Onboarding Self-Service Portal for ShopEasy 
ecommerce marketplace. This project demonstrates 
complete BA lifecycle from problem identification 
to requirement sign-off covering the most complex 
and comprehensive BA deliverables.

## Business Problem
ShopEasy's manual seller onboarding process was 
taking 2-3 weeks per seller with 40% seller drop-off 
rate and 35% document error rate. The Operations team 
spent 60% of their time on manual data entry instead 
of strategic activities causing estimated Rs. 85 lakh 
monthly GMV loss from sellers abandoning the process.

## Business Goals
| Goal | Current | Target |
|------|---------|--------|
| Onboarding time | 2-3 weeks | 3 business days |
| Seller drop-off rate | 40% | Below 15% |
| Document error rate | 35% | Below 5% |
| Operations manual effort | 60% | Below 20% |
| New seller activations/month | 120 | 350 |
| Seller satisfaction score | 2.8 / 5.0 | Above 4.2 / 5.0 |

## What I Did as BA

### Stakeholder Analysis
- Identified 9 stakeholder groups including external
  Government API providers
- Mapped influence and interest for each stakeholder
- Created complete RACI Matrix across all project
  activities defining Responsible, Accountable,
  Consulted, and Informed roles

### AS-IS Process Documentation
- Documented complete 9-step manual onboarding process
- Identified pain points at each step with time taken
- Quantified business impact — 2-3 weeks and 40% drop-off

### TO-BE Process Design
- Designed 5-step digital self-service process
- Reduced onboarding from 3 weeks to 3 business days
- Eliminated all manual handoffs between teams

### Gap Analysis
- Performed 10-area gap analysis between AS-IS and TO-BE
- Identified all systems and processes that needed change
- Mapped each gap to specific build requirements

### Requirements Documentation
- 6 seller registration requirements
- 7 document upload requirements
- 6 automated verification requirements
- 5 contract signing requirements
- 4 account activation requirements
- 4 application status tracking requirements
- 5 non-functional requirements
- Total: 37 requirements across 7 groups

### User Stories
- 9 user stories with complete Given/When/Then criteria
- Covered self-registration, business details, document
  upload, rejection handling, GST/PAN verification,
  bank verification, contract signing, account
  activation, and status tracking

### RTM
- Mapped all 37 requirements to user stories
- Tracked development fix IDs for all requirements
- Documented test status across SIT, UAT, Production
- 100% SIT pass rate
- 92% UAT pass rate
- 3 open defects documented with severity and status

## Project Scope
**In Scope:**
- Seller self-registration with OTP verification
- Digital document upload — GST, PAN, Bank Cheque, Address Proof
- Automated GST and PAN verification via government APIs
- Bank account verification via penny drop method
- Digital contract signing via eSign integration
- Automated seller account creation on approval
- Application status tracking portal

**Out of Scope:**
- Mobile app — Phase 2
- International seller onboarding
- Bulk seller import
- Seller training content

## Glossary Highlights
- **GSTIN** — 15-digit GST identification number
- **Penny Drop** — Bank verification by crediting Re.1
- **eSign** — Legally valid digital signature
- **GMV** — Gross Merchandise Value
- **RACI** — Responsible Accountable Consulted Informed
