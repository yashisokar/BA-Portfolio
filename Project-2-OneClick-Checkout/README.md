# Project 2 — One Click Checkout BRD (ShopEasy Ecommerce)

## Overview
Complete Business Requirement Document for implementing 
a One Click Checkout feature for ShopEasy — a retail 
ecommerce platform serving 2 million+ customers across India.

## Business Problem
ShopEasy's current 6-step checkout process was causing 
68% cart abandonment rate among returning customers 
resulting in estimated Rs. 2.4 crore monthly revenue loss. 
Returning customers with saved address and payment details 
were being forced to re-enter information repeatedly 
creating unnecessary friction.

## Business Goals
| Goal | Current | Target |
|------|---------|--------|
| Cart abandonment rate | 68% | Below 35% |
| Checkout conversion | 32% | Above 65% |
| Checkout time | 4-6 minutes | Under 30 seconds |
| Repeat purchase rate | 28% | Above 45% |
| Customer satisfaction | 3.6 / 5.0 | Above 4.3 / 5.0 |

## What I Did as BA

### Stakeholder Analysis
- Identified 10 stakeholder groups
- Mapped influence and interest for each
- Created stakeholder communication plan

### AS-IS Process Documentation
- Documented current 6-step checkout process
- Identified pain points at each step
- Quantified business impact of each pain point

### TO-BE Process Design
- Designed new 1-click checkout flow
- Reduced customer steps from 6 to 3
- Built order preview popup to prevent accidental orders

### Gap Analysis
- Compared AS-IS vs TO-BE across 7 areas
- Identified all changes required
- Mapped each gap to specific requirements

### Requirements Documentation
- 10 core One Click Checkout requirements
- 5 saved address management requirements
- 6 saved payment method requirements
- 6 non-functional requirements
- Total: 27 requirements with MoSCoW prioritization

### User Stories
- 8 user stories with complete Given/When/Then acceptance criteria
- Covered happy path, alternate flow, and error scenarios
- Stories for OCC from product page, cart, address change,
  payment change, cancellation, address management,
  payment management, and payment failure handling

### RTM
- Mapped all 27 requirements to user stories
- Tracked development fix IDs
- Documented test status across SIT, UAT, Production
- 100% SIT pass rate
- 96% UAT pass rate
- 1 open defect documented with severity and status
